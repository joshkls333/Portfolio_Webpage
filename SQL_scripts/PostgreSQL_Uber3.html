Count completed trips in Manila (city_id = 200) which were requested in the last year and were done by drivers who signed up in Manila. Use column request_timestamp_local and show the result as trip_count.

 

SELECT

  count(uuid) as trip_count

FROM fact_trip t

JOIN dim_driver d

  ON d.driver_uuid = t.driver_uuid

WHERE city_id = 200

  AND is_completed IS TRUE

  AND request_timestamp_local between now() - interval '1 year' and now()

  AND signup_city_id = 200

For each city in the region US West, show:

its name,

the number of distinct trips which have the status 'completed' and have been requested in the last 3 months local time (name the column as completed),

the number of all trips requested in the last 3 months local time (show as all)

and the ratio defined as the division of those 2 values (name it ratio).

When you calculate the last column, remember you need to repeat the expressions you used in the previous columns you can't use aliases.

 

SELECT

  c.city_name,

  count(DISTINCT case when t.status = 'completed' THEN t.uuid END) as completed,

  count(t.uuid) as all,

  count(DISTINCT case when t.status = 'completed' THEN t.uuid END)/count(t.uuid) as ratio

FROM fact_trip t

JOIN dim_city c

  on c.city_id = t.city_id

WHERE REGION = 'US West'

  and request_timestamp_local between now() - interval '3 months' and now()

GROUP BY 1

For each week of December 2015 local time, show: the week itself (as column week) together with the count of trips which began in this week, where trip_distance_mileswas below 5 and the client had signed up in a city different than San Francisco.

 

SELECT

  date_trunc('week', begintrip_timestamp_local) as week,

  COUNT(uuid)

FROM fact_trip t

JOIN dim_client c

  ON c.user_uuid = t.client_uuid

WHERE trip_distance_miles < 5

  AND c.signup_city_id != 1

  AND begintrip_timestamp_local between '2015-12-01' and '2015-12-01'::date + interval '1 month'

GROUP BY 1

 

Write the proper query to answer the following question: what's the name of a city which had the most requested and completed trips on December 31, 2015 UTC time?

Got stuck? Here's a hint!

First, write the ranking query. You have to join tables fact_trip and dim_city. Rank by the number of trips (count) in the descending order to get the city with most trips at rank 1.

 

SELECT

  city_name

FROM

(SELECT

  RANK() OVER (ORDER BY COUNT(t.uuid) DESC) AS rank,

  c.city_name,

  count(t.uuid) as trips

FROM fact_trip t

JOIN dim_city c

  ON t.city_id = c.city_id

WHERE t.status = 'completed'

 AND request_timestamp_utc between '2015-12-31' and '2015-12-31'::timestamp + interval '1 day'

GROUP BY city_name    ) AS a

WHERE rank = 1
